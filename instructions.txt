## 5. 3D & MODERN SCROLL EXPERIENCE (CORE 2026 TREND LAYER)

This section defines how the site should feel "alive" and current, matching trends on Awwwards/FWA-tier agency sites.

### 5.1 3D Direction
- Use a single signature 3D element system (not random 3D everywhere) — an abstract geometric object, particle field, or product-representative shape that appears in the Hero and reappears (transformed) at key trust/CTA moments, creating narrative continuity.
- Recommended stack: Three.js / React Three Fiber (R3F) for full control, or Spline for faster iteration when a non-engineer needs to edit the scene.
- The 3D object should respond to: scroll position (rotate/morph/move along a path), cursor movement (subtle parallax tilt, magnetic pull — never disorienting), and section transitions (handing off the story from one section to the next).

### 5.2 Scroll Experience
- Use smooth/inertia scrolling (Lenis or Locomotive Scroll) so the site feels like one continuous surface, not default browser scroll.
- Use scroll-triggered choreography (GSAP ScrollTrigger or Framer Motion useScroll) for: text reveal (stagger), section pinning (max 1–2 per page), horizontal scroll reveals (only when it improves scanability), and scroll-linked progress indicators.
- Parallax layered by depth (background slowest, foreground fastest), kept subtle (max ~15–20% offset) to avoid motion sickness or performance drops.

### 5.3 Micro-Interactions
Magnetic buttons, smooth eased hover states, animated counters (trigger once on view), cursor-aware card glows, fast page transitions (<400ms, fade + slight scale).

### 5.4 Non-Negotiable Performance & Accessibility Rules
- Mobile fallback is mandatory: heavy 3D/WebGL must degrade to a static image, lightweight animation, or CSS-only version on mobile/low-end devices. Never ship a laggy 3D scene to mobile.
- Respect prefers-reduced-motion.
- Lazy-load all 3D/WebGL bundles; load only when the relevant section nears the viewport.
- Every animation must guide attention, show relationship, or reinforce hierarchy — never animate to look busy.
- Target: Hero interactive within 2.5s even on mid-tier mobile. If 3D can't hit this, defer it behind a lightweight poster frame.
- "Never over-animate" is a hard rule — when in doubt, remove an animation rather than add one.

## 8. INTERIOR PAGE ARCHITECTURE (CRITICAL SECTION)

The homepage sets the visual language. Every interior page inherits it exactly (navbar, footer, typography, spacing, buttons, color, motion easing) — but each page type has its own content logic and structure, not a copy of the homepage's section order.

### 8.1 Service Page (Website Development, UI/UX Design, API Development, etc.)
1. Focused hero — one service, one promise, a quieter variation of the signature 3D object
2. Problem this service solves
3. What's included (feature grid)
4. Process for this specific service (short timeline)
5. Relevant tech stack
6. 1–2 relevant case studies (not a full grid)
7. Pricing indication or "Get a quote" CTA
8. Service-specific FAQ
9. Final CTA

### 8.2 Case Study / Portfolio Detail Page
1. Hero with project name, client, one strong result stat (animated counter)
2. Challenge → Approach → Solution narrative (best place for scroll-pinned before/after transitions)
3. Visual gallery (real screenshots, lazy-loaded)
4. Tech stack used
5. Measurable outcomes
6. Client testimonial (if available)
7. Next case study link + CTA

### 8.3 About Page
1. Hero: mission statement, quiet/no heavy 3D (trust tone, not spectacle)
2. Story/why NovaDesk exists (short, human)
3. Values or working principles (3–4 cards max)
4. Team or capability showcase
5. Stats (animated counters)
6. CTA

### 8.4 Pricing / Packages Page
1. Hero: clear value framing, no 3D distraction
2. Plan comparison (table or cards, scannable)
3. What's included per tier
4. FAQ (billing, timelines, revisions, code ownership)
5. "Not sure which plan?" CTA → consultation

### 8.5 Contact Page
1. Short, confident hero, minimal motion
2. Direct contact form (minimal fields, smart validation, clear success state)
3. Alternative contact methods
4. Optional: location / hours / response-time expectation
5. Small trust reinforcement (logos, stat, or one testimonial line)

### 8.6 Blog / Insights (if included)
1. Clean list/grid, no 3D — readability and scan-speed matter most
2. Category filters
3. Article template: strong typography, minimal chrome, reading-progress indicator

Rule of thumb: the deeper the page is in the funnel (Pricing, Contact), the less spectacle and the more clarity/speed. The 3D/motion system is front-loaded at awareness stages (Homepage, Service intros, Case studies) and quieted at decision stages so nothing slows down the enquiry.

## 10. COMPONENT SYSTEM

Reuse components: Navbar, Footer, Buttons, Cards, Forms, Icons, Spacing tokens, Typography scale, Grid, Badges, Inputs, and the Signature 3D Object component (parameterized: size, intensity, scroll-behavior variant). Everything stays visually and behaviorally consistent; only page-specific content changes.

---

## 11. RESPONSIVE DESIGN

Desktop-first, then Tablet, then Mobile. Perfect spacing/typography at every breakpoint. Touch-friendly targets (min 44px). 3D/heavy motion scales down or swaps to lightweight fallback below tablet breakpoint (see 5.4).

---

## 12. TECHNICAL REQUIREMENTS

React, modular components, no duplicated code, UI separated from logic. React Three Fiber or Spline for 3D; GSAP/Framer Motion for scroll and micro-interactions; Lenis for smooth scroll. Code-split and lazy-load heavy assets. Semantic HTML, accessibility-friendly, reduced-motion respected. SEO-friendly. Optimize Core Web Vitals — 3D/motion must never push LCP/CLS/INP into poor territory.

---

## 13. PROCESS — WHEN GENERATING ANY PAGE

Do not immediately write JSX/code:
1. Analyse the page's objective and funnel position (awareness vs. decision stage — see Section 8)
2. Analyse the supplied content
3. Create the best information hierarchy
4. Design the user flow
5. Choose visual components and decide how much 3D/motion this page deserves
6. Only then build the page

Always explain reasoning internally before generating implementation. The final result must feel handcrafted by an elite product design agency — impressive enough that clients instantly trust NovaDesk and feel confident reaching out, while every inside page stays fast, clear, and true to the same design system as the homepage.
